# План приложения для создания страниц сайта на Tilda

## 1. Исследование (MCP Context7 и документация)

### Результаты анализа

- **MCP Context7**: найдена актуальная документация Tilda:
  - `/websites/help_tilda_cc` — Tilda Help Center (Benchmark 73.9)
  - `/websites/help-ru_tilda_cc` — Tilda Publishing (2142 сниппетов)
  - `/websites/help_tilda_cc_zero` — Tilda Zero Block (кастомные блоки)

- **Официальный Tilda API** ([help.tilda.cc/api](https://help.tilda.cc/api)):
  - Доступен только на тарифе **Business Plan** (Public Key + Secret Key в Настройки сайта → Экспорт → API).
  - **Ограничение**: API только для **чтения и экспорта**. Эндпоинтов для **создания** новых страниц нет.
  - Доступные методы: `getprojectslist`, `getprojectinfo`, `getpageslist`, `getpage`, `getpagefull`, `getpageexport`, `getpagefullexport`.
  - Лимит: 150 запросов в час. Контент нужно сохранять на свой сервер, не дергать API при каждом визите пользователя.
  - Есть **Webhook** при нажатии «Опубликовать» (callback URL в настройках API).

- **Вывод**: приложение не может «создать страницу в Tilda» через API. Реалистичные сценарии:
  1. **Визуальный конструктор страницы** (блоки в стиле Tilda/Zero Block) с экспортом в HTML.
  2. **Интеграция с Tilda API** для просмотра проектов/страниц и экспорта уже созданных в Tilda страниц на свой сервер или в приложение.

## 2. Цели и границы MVP

| Цель | Описание |
|------|----------|
| Конструктор страницы | Визуальное создание страницы из блоков (заголовок, текст, изображение, кнопка, форма, колонки). |
| Экспорт | Экспорт в HTML (standalone или совместимый со стилями Tilda) и при необходимости JSON-схема блоков для последующего импорта. |
| Интеграция с Tilda (опционально) | Для пользователей с Business Plan: список проектов, список страниц, просмотр и экспорт страницы (getpage / getpageexport). |
| Публикация плана и кода | Репозиторий [Horosheff/Tilda](https://github.com/Horosheff/Tilda) — план, README, структура проекта, MVP. |

## 3. Функциональные блоки MVP

### 3.1 Визуальный редактор страницы

- Холст с drag-and-drop блоков.
- Типы блоков (минимум для MVP):
  - Заголовок (H1–H3)
  - Текст (параграф)
  - Изображение (URL или загрузка)
  - Кнопка (текст, ссылка, стиль)
  - Блок «Колонки» (2–3 колонки)
  - Форма (имя, email, сообщение — базовая разметка)
- Редактирование свойств блока в панели справа (или инлайн).
- Предпросмотр страницы в реальном времени (desktop / mobile).

### 3.2 Данные и экспорт

- Внутреннее представление страницы: **JSON** (дерево блоков с типами и полями).
- Экспорт:
  - **HTML** — одна страница с инлайновыми или подключёнными CSS, готовая к вставке в Tilda (body) или к самостоятельному размещению.
  - Опционально: **JSON** — сохранение/загрузка проекта в приложении.

### 3.3 Интеграция с Tilda API (опционально в MVP)

- Экран «Подключить Tilda»: ввод Public Key и Secret Key (хранение только на клиенте, без отправки на чужой сервер).
- Запросы к `https://api.tildacdn.info/`:
  - Список проектов → выбор проекта.
  - Список страниц проекта → выбор страницы.
  - Получение страницы (`getpage` / `getpageexport`) и отображение HTML или парсинг в блоки (если решим поддерживать импорт).
- Учёт лимита 150 req/h и предупреждение пользователю.

### 3.4 Улучшения (после MVP)

- Использование документации **Zero Block** из Context7 для более близкой к Tilda разметки и классов.
- Импорт страницы из Tilda (HTML → разбор в блоки редактора).
- Шаблоны страниц (лендинг, контакты, портфолио).
- Темы оформления (цвета, шрифты) и экспорт CSS.
- Сохранение проектов в облаке (backend + авторизация).

## 4. Технологический стек (рекомендация)

| Слой | Вариант |
|------|---------|
| Frontend | React + TypeScript (или Vue 3 + TS) |
| Сборка | Vite |
| Состояние редактора | Zustand или React Context |
| Drag-and-drop | @dnd-kit/core или react-beautiful-dnd |
| Стили | Tailwind CSS + CSS-переменные для тем |
| Сеть (Tilda API) | fetch + обёртка с ключами и лимитами |
| Экспорт HTML | Рендер по JSON-дереву блоков в строку HTML |

Бэкенд для MVP не обязателен: всё в браузере, ключи Tilda только на клиенте.

## 6. Этапы реализации до MVP

1. **Инициализация проекта**: Vite + React + TypeScript + Tailwind, структура папок, базовый роутинг (если нужен).
2. **Модель данных**: типы `Block`, `Page`, хранилище (Zustand/Context), добавление/удаление/перемещение блоков.
3. **Редактор**: холст, список блоков с DnD, панель свойств для выбранного блока.
4. **Блоки**: реализация 5–6 типов блоков (заголовок, текст, изображение, кнопка, колонки, форма) и их рендер в редакторе и в превью.
5. **Экспорт**: генерация HTML из дерева блоков, кнопка «Скачать HTML», опционально «Скачать JSON».
6. **Tilda API**: экран настроек ключей, запросы getprojectslist, getpageslist, getpage/getpageexport, отображение списка и HTML выбранной страницы.
7. **Доработки**: адаптивное превью, базовая стилизация под «стиль Tilda», README с инструкцией, публикация в [Horosheff/Tilda](https://github.com/Horosheff/Tilda).
